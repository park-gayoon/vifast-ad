HTML

Syntax

  • indent : 탭(tab)을 사용
  • attribute에는 double Quotation("")를 사용
  • Less Than(<), Greater Than(>), Ampersand(&) 기호에 html 엔터티를 사용

doctype

doctype은 html5로 지정 <!DOCTYPE html>

language 속성

<html lang="ko">

CSS

Syntax

  • indent : 탭(tab)을 사용
  • 여러 셀렉터를 하나에 쓸 경우, 한 중에 하나의 셀렉터 사용
    ex) margin-left:0;margin-top:10px (X) >> margin:10px 0 0 (O)
  • {앞뒤에 한칸 띄우지 않음}
  • :뒤에 한칸 띄우지 않음
  • ;뒤에 한칸 띄우지 않음
  • 마지막 ; 삭제함
  • 콤마(,) 다음에 한칸 띄우기
  • rgb(), rgba(), hsl(), hsla(), rect()안의 콤마(,)는 띄어쓰기 하지 않음
  • 소수점 0으로 시작하는 value는 0을 생략 0.5대신 .5사용
  • 속성 값이 '0'이면 단위를 생략

선언순서

선언의 순서는 아래와 같이 그룹핑

  1. 1. 위치관련(position, top, right, bottom, left, z-index)
  2. 2. 박스모델 관련(display, float, width, height, margin, padding)
  3. 3. 타이포그래피(font, line-height, color, text-align)
  4. 4. 시각효과(background, border)
  5. 5. 기타(opacity)

@import 사용 지양

css내에 @import는 성능을 저하시키므로 여러개의 css를 link

id/class 사용

레이아웃을 제외하고 가급적 id 사용은 지양

css 파일 구조

  • common.css: html 기본 속성값 초기화/공통 설정값(H tag, Button, Form, table, ul list)
  • contents.css: 서브 컨텐츠
  • main.css: 메인 전용
  • modal_style.css: 모달 전용(개발 플러그인 css)

id/class 명명 규칙

  • _(언더바)를 사용한다.
  • id는 camelcase 방식을 사용하고 class는 underscore 방식을 사용한다.
    ex) id="boxWrap" class="small_box"
  • id, class naming 은 가능하면 중복되지 않게 생성한다.
  • naming은 형태,의미,상태 순서로 조합하며 3단계를 넘어가지 않도록 한다. 3단계가 넘어갈 경우 -(하이픈)을 사용한다
    ex) btn_modify-on
  • 같은 형태에서 타입이 다른 케이스가 나오는 경우 멀티 class를 사용한다
    ex) class="bullet_list" class="bullet_list type_a"

Image

image 폴더 구조

  • common: header, footer 영역과 같이 공통으로 사용되는 이미지
  • main: 메인에 사용되는 이미지
  • contents: 서브 페이지에 사용되는 이미지
  • temp: 실제 반영되지 않을 샘플 이미지

image 명명 규칙

  • -(하이픈)은 가능하면 사용하지 않고 _(언더바)를 사용한다.
